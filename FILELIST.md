# FILELIST.md

- `.gitignore` ‚Äî Git ignore rules.
- `.gitattributes` ‚Äî Marks binary assets (`.syso`, `.ico`, `.icns`, `.png`, `.zip`) so Git never applies line-ending conversion that would corrupt them.
- `go.mod` ‚Äî Go module definition file.
- `.github/workflows/release.yml` ‚Äî GitHub Actions: on a `v*` tag, builds the universal macOS `.app` (version injected) and the Windows `setup.exe` installer, publishes them to a GitHub Release, and bumps the multi-claude-switcher cask in `miou1107/homebrew-tap` to the new version + macOS zip SHA256 (needs the `HOMEBREW_TAP_TOKEN` secret).
- `go.sum` ‚Äî Go module checksum file.
- `README.md` ‚Äî Project overview, architecture, and quick start CLI/GUI guide (English).
- `README.zh-TW.md` ‚Äî Traditional Chinese translation of the README.
- `FILELIST.md` ‚Äî Inventory of project files and purpose.
- `CHANGELOG.md` ‚Äî Project version and change history.
- `bin/mcs` ‚Äî Compiled CLI binary executable.
- `bin/mcs-tray` ‚Äî Compiled System Tray GUI binary executable.
- `cmd/mcs/main.go` ‚Äî Go CLI entry point (`mcs status`, `sync`, `switch`, `backup`, `restore`).
- `cmd/mcs-tray/main.go` ‚Äî Go System Tray GUI entry point (cross-platform menu-bar / tray quick switcher).
- `cmd/mcs-tray/trayicon_darwin.go` ‚Äî macOS tray icon (template PNG via SetTemplateIcon).
- `cmd/mcs-tray/trayicon_windows.go` ‚Äî Windows tray icon (icon.ico via SetIcon).
- `cmd/mcs-tray/trayicon_other.go` ‚Äî Tray-icon no-op for unsupported OSes.
- `cmd/mcs-tray/assets/icon.png` ‚Äî Menu bar template icon (eyes glyph; recolored by macOS for light/dark).
- `cmd/mcs-tray/assets/appicon-1024.png` ‚Äî Color 1024px app icon source (eyes), converted to icon.icns at packaging time.
- `cmd/mcs-tray/assets/icon.ico` ‚Äî Color multi-resolution Windows icon (16‚Äì256px) for the tray/exe.
- `cmd/mcs-tray/rsrc_windows_amd64.syso` ‚Äî Windows resource object (generated from `icon.ico` via `rsrc`) that compiles the app icon into `mcs-tray.exe`; auto-linked for windows/amd64 so the Start Menu / taskbar / Explorer show the icon.
- `scripts/gen-icons/main.go` ‚Äî Standalone generator (`go run`) that rasterizes the eyes mark from geometry into all icon assets (app PNG, menu-bar template, Windows .ico, doc PNG); no external tools needed.
- `docs/assets/icon.png` ‚Äî 512px color icon for README and documentation.
- `core/version.go` ‚Äî Single source of truth for the product version (imported by CLI and tray).
- `core/logging.go` ‚Äî Persistent per-component logging to ~/.multi-claude-switcher/logs (stderr + file).
- `core/names.go` ‚Äî User-chosen profile display names, stored in ~/.multi-claude-switcher/names.json.
- `core/loginitem_darwin.go` ‚Äî Start-at-login LaunchAgent management on macOS (install/remove per-user plist).
- `core/loginitem_windows.go` ‚Äî Start-at-login on Windows via the per-user registry Run key (HKCU\...\Run).
- `core/hidewindow_windows.go` ‚Äî Windows helper: run the `reg` login-item commands with CREATE_NO_WINDOW so no console window flashes.
- `core/loginitem_other.go` ‚Äî Start-at-login stub for non-macOS/Windows builds (safe no-ops).
- `core/loginitem_darwin_test.go` ‚Äî Unit tests for macOS login-item enable/disable and plist contents (stubbed dir).
- `core/loginitem_windows_test.go` ‚Äî Unit tests for Windows login-item enable/disable via a throwaway registry key.
- `core/update.go` ‚Äî Update check against GitHub Releases (version compare, latest-release fetch, download).
- `core/update_test.go` ‚Äî Unit tests for version comparison and release JSON parsing.
- `cmd/mcs-tray/update.go` ‚Äî Update check plumbing shared by all OSes: find the platform's release asset, single-flight the check, periodic + manual triggers; delegates the actual install to the per-OS `installUpdate`.
- `cmd/mcs-tray/update_test.go` ‚Äî Unit tests for .app-bundle path detection and release-asset matching.
- `cmd/mcs-tray/update_install_nonwindows.go` ‚Äî macOS/Unix `installUpdate`: download the `.app` zip, atomically swap the tray binary, relaunch (bundle-aware).
- `cmd/mcs-tray/update_install_windows.go` ‚Äî Windows `installUpdate`: notify of a new version and open the download page (the setup.exe upgrades in place); no in-place binary swap.
- `cmd/mcs-tray/update_platform_darwin.go` ‚Äî macOS self-update helpers: `_macos.zip` suffix, ditto extraction, quarantine strip, .app binary lookup.
- `cmd/mcs-tray/update_platform_windows.go` ‚Äî Windows release-asset suffix (`_windows_setup.exe`); the installer is the update signal.
- `cmd/mcs-tray/update_platform_other.go` ‚Äî Self-update stubs for unsupported OSes.
- `cmd/mcs-tray/instance.go` ‚Äî Single-instance guard: shared skip-flag check + pure ps-output tray parser, plus the relaunch exemption flag.
- `cmd/mcs-tray/instance_unix.go` ‚Äî macOS/Unix instance check (ps scan).
- `cmd/mcs-tray/instance_windows.go` ‚Äî Windows instance check (tasklist filtered to mcs-tray.exe) and its pure parser.
- `cmd/mcs-tray/instance_test.go` ‚Äî Unit tests for the skip-flag check and the ps-output tray-detection parser.
- `cmd/mcs-tray/instance_windows_test.go` ‚Äî Unit tests for the Windows tasklist tray-detection parser.
- `cmd/mcs-tray/relaunch_unix.go` ‚Äî Self-update relaunch detach on Unix (own process group via Setpgid).
- `cmd/mcs-tray/relaunch_windows.go` ‚Äî Self-update relaunch detach on Windows (CREATE_NEW_PROCESS_GROUP).
- `cmd/mcs-tray/hidewindow_windows.go` ‚Äî Windows helper: launch a spawned console helper (powershell/tasklist) with CREATE_NO_WINDOW so no console window flashes.
- `cmd/mcs-tray/profiles_windows.go` ‚Äî Windows-only tray flow: "New account profile‚Ä¶" (Store build) ‚Äî save the current account and open a fresh Claude to sign into another, then relaunch the tray.
- `cmd/mcs-tray/profiles_other.go` ‚Äî Non-Windows no-ops for the new-profile flow (macOS standalone profiles are ordinary sibling data dirs).
- `packaging/Info.plist.template` ‚Äî macOS bundle Info.plist template (LSUIElement agent; version substituted at build).
- `packaging/windows-setup.iss` ‚Äî Inno Setup script for the Windows installer (per-user install, Start Menu shortcut, uninstaller).
- `scripts/package-app.sh` ‚Äî Assembles Multi-Claude Switcher.app (binary + Info.plist + icon), ad-hoc signs it (`codesign --sign -`, no Apple Developer account needed), and zips it via ditto.
- `core/backup.go` ‚Äî Profile backup & snapshot restoration module (atomic restore).
- `core/backup_test.go` ‚Äî Unit tests for backup & restore manager.
- `core/backup_perm_unix_test.go` ‚Äî Test helper (non-Windows): make a dir reject new children via chmod, to exercise staging-failure paths.
- `core/backup_perm_windows_test.go` ‚Äî Test helper (Windows): make a dir reject new children via an icacls deny ACE, to exercise staging-failure paths.
- `core/sync.go` ‚Äî Session index synchronization module with conflict detection, plus `SyncBidirectional` (unions both profiles' sessions for auto sync).
- `core/sync_test.go` ‚Äî Unit tests for session sync (copy, conflict, identical, overwrite).
- `core/switch.go` ‚Äî Safe Switch controller (Terminate -> Backup -> Sync -> Launch); session data only moves when the auto sync toggle is on (bidirectional align), otherwise a switch is a pure account change.
- `core/switch_test.go` ‚Äî Unit tests for Safe Switch (aborts on backup failure).
- `core/settings.go` ‚Äî User settings store (~/.multi-claude-switcher/settings.json): auto sync toggle + warning-dismissed flag.
- `core/settings_test.go` ‚Äî Unit tests for settings round-trip, defaults, and no-clobber.
- `core/align.go` ‚Äî Manual directional align (ManualAlign): close ‚Üí backup ‚Üí sync ‚Üí reopen the same account.
- `core/align_test.go` ‚Äî Unit tests for ManualAlign (returns to running profile; no relaunch when nothing ran).
- `cmd/mcs-tray/autosync.go` ‚Äî Tray Auto Sync toggle: enable-time warning dialog and choice parsing.
- `cmd/mcs-tray/autosync_test.go` ‚Äî Unit tests for the auto-sync warning-gating helper.
- `cmd/mcs-tray/dialog_darwin.go` ‚Äî macOS tray dialogs / notifications (osascript).
- `cmd/mcs-tray/dialog_windows.go` ‚Äî Windows tray dialogs / notifications (PowerShell + WinForms).
- `cmd/mcs-tray/dialog_other.go` ‚Äî Dialog stubs for non-macOS/Windows builds (no-ops).
